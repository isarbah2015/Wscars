Root Cause — Confirmed
ensureReady() in your service layer calls isFirebaseReady() which checks app !== null && db !== null — but never checks auth. Meanwhile auth! is used with a non-null assertion. If auth is null, TypeScript trusts the ! and passes null to Firebase functions which silently no-ops or throws an uncaught error that AppContext never catches.
The chain: auth can be null → ensureReady() doesn't catch it → signInWithEmailAndPassword(null!, ...) → silent failure.

Fix 4 — app/_layout.tsx — fix cold-open redirect
Replace useAuthRedirect with this:
typescriptfunction useAuthRedirect() {
  const segments = useSegments();
  const router = useRouter();

  useEffect(() => {
    const unsub = onAuthStateChanged(auth, (user) => {
      const isAuthed = !!user;
      const onAuthScreen =
        segments.includes('login') ||
        segments.includes('signup') ||
        segments.includes('welcome') ||
        segments.length === 0; // splash / initial load

      if (isAuthed && onAuthScreen) {
        // Logged in but on auth screen — always redirect (handles cold open)
        router.replace('/(tabs)');
      } else if (!isAuthed && !onAuthScreen) {
        // Not logged in but inside the app — push back to welcome
        router.replace('/welcome');
      }
      // Otherwise stay put
    });
    return unsub;
  }, [segments]); // segments in deps so it re-evaluates on navigation
}
Key change: removed prevAuthRef entirely. The old logic only redirected on transition. This version redirects based on current state on every auth check — so cold open with a restored session correctly pushes to tabs immediately.
